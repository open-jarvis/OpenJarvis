# Data Sharing Policy Summary for Serena

Status: local_reference_v1
Policy area: Drive, Docs, email, exports, browser, uploads

Purpose:
Serena must guard external sharing and storage of sensitive data.

Core Serena rules:
- Before upload/share/export, classify the content.
- If content contains personal, patient, health, biometric, financial, or business-sensitive data, report risk.
- Use explicit reporting for Google Drive, Google Docs, Calendar, OCR, CRM, and finance actions.
- Return links without making files public unless explicitly requested and safe.

Blocked:
- Silent uploads of sensitive information.
- Secret exposure.
- Public sharing of patient/health data without authorization.
- Destructive bulk export or disclosure.
